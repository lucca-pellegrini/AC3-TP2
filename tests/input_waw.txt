# WAW hazard demo: a slow MUL.D writes F1, then a fast ADD.D
# (issued later in program order) also writes F1.
#
# Without register renaming a naive pipeline would see the ADD finish
# first and then have the late MUL clobber it, producing the WRONG value.
# Tomasulo's algorithm uses the ROB + RAT so that commits happen in
# program order: the ADD is the *last* writer in program order, so the
# final value of F1 must be 3 (the ADD's result), not 20 (the MUL's).
#
# A third instruction reads F1 and must see that latest value: F6 = 3 + 1 = 4.

CONFIG_BEGIN

CYCLES ADDD 2
CYCLES MULTD 10

UNITS ADDD 2
UNITS MULTD 1

CONFIG_END

REG_INIT_BEGIN
F2 1.0
F3 2.0
F4 4.0
F5 5.0
F7 1.0
REG_INIT_END

INSTRUCTIONS_BEGIN
MULTD F1 F4 F5    # F1 = 4*5 = 20  (slow, finishes late)
ADDD  F1 F2 F3    # F1 = 1+2 = 3   (fast, overwrites in program order)
ADDD  F6 F1 F7    # F6 = F1 + F7 -> must read the *later* F1 = 3, so F6 = 4
INSTRUCTIONS_END
